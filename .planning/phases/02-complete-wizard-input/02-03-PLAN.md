## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|--------------|-----------------|
| T-02-04 | Tampering | app/api/assessments/route.ts | mitigate | Every array field (departments[].capabilities, techStack.systems, aiAdoption.toolsInUse, automation.processesAutomated, agentInterest.agentTypes, security.checklist) is validated as a subset of its known option-list constant with no duplicate entries, preventing forged option values or ratio-inflating duplicate entries from reaching the scoring engine |
| T-02-05 | Tampering | app/api/assessments/route.ts (numeric ratings) | mitigate | departments[].rating, dataReadiness.*, employeeReadiness.* are all validated as finite numbers in the inclusive 0-5 range before being scored or persisted, rejecting NaN/Infinity/out-of-range/string-typed values with a 400 |
| T-02-06 | Denial of Service | app/api/assessments/route.ts | accept | No explicit request-body size cap is added in this plan -- Next.js's default body parser limits and the subset/no-duplicate validation above already bound array sizes to each option list's known length; full request-size limiting is out of scope for this local-only, single-user v1 tool per PROJECT.md constraints |
