<objective>
Implement ENG-04 (stage ladder guidance text) and ENG-05 (data-completeness
signal) as two independent, pure, fully-tested functions per the design
spec's "Scoring engine" section: extend the existing 6-band `getStage`
lookup to also return a static "what this means" / "what's next" sentence
per stage, and add a `computeDataCompleteness` function implementing the
spec's "fewer than half the optional checklist items touched" rule.

Purpose: Both are pure functions with well-defined inputs/outputs and real
risk of subtly wrong text/math if untested — the canonical TDD case. Building
them here, independent of any wiring, lets Plan 03-03 wire both into the
results page and API route without touching this plan's files.
Output: `getStage` returning the full `Stage` shape; `lib/scoring/completeness.ts`
(new) exporting `computeDataCompleteness` — both proven by passing tests.
</objective>
